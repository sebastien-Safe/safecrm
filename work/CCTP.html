Propositions UX/UI

Propositions de refonte de /work/index.html uniquement. Aucun code produit à ce stade.

🛑 STOP — Ces propositions attendent votre validation explicite avant tout développement.

Proposition A — Dashboard modernisé (évolution douce)

Conserver la structure existante, améliorer le rendu

Même sidebar, mêmes vues. Moderniser les cartes stats avec micro-animations, ajouter un graphique de tendance, afficher les modules avec leur état temps réel. Faible risque de régression.

Ajouter une vue "Automatisations actives"

Nouvelle vue listant les automatisations configurées (data-driven depuis Supabase), pas hardcodées. Nécessite table work_automations. Remplace l'objet JS WORKFLOWS.

Migrer le journal vers Supabase

Créer table work_journal (id, user_id, wf_id, contact_id, action, status, created_at). Remplacer localStorage. Journal persistant, multi-utilisateurs, exportable.

Proposition B — Orchestrateur graphique (refonte ambitieuse)

🗺 Flow Builder visuel

Canvas de type n8n/Make : nœuds glissables représentant déclencheurs, conditions, actions, connecteurs. Rendu SVG ou Canvas. Composants à créer : DragNode, FlowCanvas, ConnectorPin. Effort : élevé.

📊 Dashboard temps réel

Supabase Realtime sur work_journal et tasks. Badge de notification vivant, flux d'activité en temps réel. Effort : moyen.

🔌 Hub connecteurs intégré

Intégrer connecteurs.html comme vue interne de /work plutôt que page séparée. Vue "Connecteurs" dans la sidebar. Effort : faible.

📁 Factoriser le CSS partagé

Créer /work/shared.css avec les composants communs (.card, .btn, .modal, .spinner, .toast, .badge). Chaque module l'importe + son CSS accent. Élimine 6× les doublons. Effort : moyen.

🗑 Supprimer les 8 fichiers orphelins

Nettoyer le dépôt. Vérifier que module-dpo.html n'est pas référencé en externe avant suppression. Effort : très faible.

Vérification sécurité

Audit de sécurité

  • 🟢
    RLS activé sur toutes les tables métier
    contacts, contracts, tasks, profiles, safe_connectors, modules profiles
  • 🟢
    Auth Supabase avec JWT + refresh automatique
    Token rafraîchi toutes les 50min via scheduleTokenRefresh()
  • 🟢
    Clé anon Supabase (non secret)
    Clé anon exposée côté client — comportement normal pour Supabase. RLS protège les données.
  • 🟢
    Connecteurs : aucune connexion auto sans validation admin
    connectors-guard.js vérifie statut = 'actif' | 'simule' avant tout appel
  • 🟢
    Mode simulation empêche les vraies API calls
    isSimulated() retourne mock sans appel réseau réel
  • 🟠
    Journal en localStorage — non auditable
    100 entrées max, perdu au vidage cache, pas de multi-utilisateurs. Migration Supabase recommandée.
  • 🟠
    Pas de rate limiting frontend
    Les appels sbGet/sbPost ne sont pas throttled côté client. Supabase applique ses limites propres.
  • 🟠
    Workflows hardcodés — pas de validation des inputs
    Si les workflows deviennent data-driven, valider les payloads côté Supabase (contraintes CHECK).
  • 🟢
    noindex, nofollow sur /work
    Meta robots configuré — accès non indexé par les moteurs.
  • 🟢
    Pas d'IDOR détecté
    RLS filtre par user auth — les contacts/contracts ne sont pas accessibles sans session.
