P2 OUVERT SEC-06 — TOTP trust stocké en localStorage (contournable côté client) assets/totp/totp.js

Cause : Le timestamp de trust TOTP est stocké dans localStorage (safe_totp_trust_<userId>). Un utilisateur peut modifier la valeur via DevTools pour prolonger le trust indéfiniment.

Risque résiduel : Les 3 opérations critiques (create-user, delete-user, cancel-subscription) vérifient AAL2 côté serveur — le bypass localStorage est sans effet sur elles. Le risque concerne uniquement les opérations dont la protection est purement client-side.

localStorage bypass TOTP client

Correction : Remplacer localStorage par sessionStorage dans totp.js (lignes 136, 161, 189, 287). Le trust expire à la fermeture du navigateur. Effort : 30 min.

21/23 restantes accès non restreint
