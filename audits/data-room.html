S@FE CRM est un logiciel SaaS de gestion commerciale et de conformité réglementaire, développé et édité par S@FE SASU (SIRET : 104 699 558 00011). Le produit cible les PME françaises soumises aux obligations RGPD, NIS2 et aux exigences de l'ANSSI.

Le logiciel constitue un actif immatériel structuré, industrialisé et documenté, présentant les caractéristiques suivantes :

🏗 Architecture industrielle
SPA Vanilla JS modulaire (65 fichiers JS / 19 164 lignes), backend Supabase 100% serverless, 13 Edge Functions Deno 2.x, schéma PostgreSQL en 29 versions migratoires successives.
🔒 Sécurité de niveau entreprise
Row Level Security sur 28 tables, JWT + TOTP MFA obligatoire pour les rôles sensibles, anti-bruteforce 5 tentatives, audit trail complet dans audit_logs, chiffrement AES-256 au repos.
⚖️ Conformité réglementaire native
RGPD Art. 30 (registre des traitements intégré), NIS2 (module incidents), DPA signé Supabase + Stripe + Brevo, AIPD rédigée pour la fonction IA, purge automatique des données.
🤖 Intelligence artificielle intégrée
4 fournisseurs IA en fallback automatique (Groq · Grok · Anthropic · Mistral), assistant contextuel par module, génération PDF IA-augmentée, Edge Function call-ia v4.
💳 Monétisation opérationnelle
Intégration Stripe complète (checkout, webhook, customer portal, résiliation), liens de paiement par contrat, gestion des abonnements récurrents, PDF de commande auto-généré.
📱 Application installable (PWA)
Progressive Web App certifiée, 8 tailles d'icônes, mode offline via Service Worker, installable sur iOS/Android/Desktop, orientée tablette pour usage terrain.

Présentation de S@FE CRM

Vision

Permettre aux PME et aux indépendants français de disposer d'un outil de gestion commerciale qui intègre nativement les obligations réglementaires (RGPD, NIS2, ANSSI) sans surcoût de mise en conformité ni recours à des prestataires tiers spécialisés.

Mission

S@FE CRM agrège en une seule plateforme : la gestion des prospects et clients, la contractualisation numérique, le paiement récurrent, la conformité réglementaire proactive et l'assistance IA — le tout sans infrastructure à maintenir (SaaS 100% serverless).

Commercial terrain. Gestion de son portefeuille contacts/contrats. Accès missions prospection. Pas d'accès administration.

Manage une équipe de commerciaux. Cockpit consolidé, objectifs d'équipe, mandat DCI signé électroniquement.
🔑 Administrateur
Accès total. Gestion utilisateurs, configuration modules, tableaux de bord globaux, audit RGPD, incidents NIS2.

Différenciation concurrentielle

  • Conformité by design : le RGPD et NIS2 ne sont pas des add-ons mais des fonctionnalités cœur, développées dès l'origine.
  • IA multi-fournisseur avec fallback : aucune dépendance exclusive, continuité de service garantie si un fournisseur IA est indisponible.
  • Prospection terrain native : espace dédié tablette avec PIN, missions de vente guidées, capture de consentement RGPD immédiate.
  • Zéro infrastructure : déploiement 100% Supabase Cloud (edge network EU), aucun serveur applicatif à maintenir.
  • Force de vente intégrée : génération automatique des mandats DCI, bordereaux de commission, simulation de revenus.
  • Modules verticals métier : SEO local, DPO externalisé, cybersécurité ANSSI/CIS, social media, click & collect — extensibilité sans refactoring.

Ciblage entreprises — Préparer ma cible

Double moteur de recherche : API SIRENE officielle (data.gouv.fr, gratuit, souverain) pour le ciblage par secteur/rayon, et Google Maps Places API (Text Search) pour la recherche libre par nom ou mot-clé.

🚀 SIRENE — Ciblage sectoriel
8 boutons rapides (Restauration, Boulangerie, Boucherie, Fromagerie, Traiteur, Fleuriste, Garage, Gîte) + saisie libre. Rayon 5/10/25/50 km. API data.gouv.fr, sans clé, données souveraines françaises.
🔍 Google Places — Recherche libre
Recherche d'entreprises par nom ou activité via Google Maps Places Text Search. Proxy sécurisé Edge Function places-search — clé API jamais exposée au client. Résultats en lecture seule : aucune donnée Google stockée en base (RGPD + CGU EEE).
⏱️ Rate limiting Places API
20 recherches/mois/utilisateur · 5 000 recherches/mois global. Log RGPD conservé dans places_search_logs : requête + nombre de résultats uniquement (jamais les données des entreprises). Quota affiché en temps réel dans l'interface.
✅ Capture consentement post-audit
En fin d'audit gratuit (RGPD · SEO · Cyber) : saisie obligatoire du nom de l'entreprise, du signataire, de l'email et cochage "Consentement oral obtenu". Saisie 100% manuelle — aucun import depuis Google Places. Création fiche CRM conditionnée au consentement.
⚠️ Garde-fou "À compléter"
Toute fiche créée est marquée "À compléter" (badge orange, pipeline kanban). Création de contrat bloquée jusqu'à validation manuelle complète. Bannière permanente sur les résultats Google : "Données Google — Lecture seule — Ne pas recopier dans le CRM".

Documentation Technique

Frontend — Architecture SPA

S@FE CRM est une Single Page Application construite en Vanilla JS ES2022 sans framework (ni React, ni Vue, ni Angular). Ce choix délibéré offre :

  • Zéro dépendance npm en production : aucune chaîne de build complexe, aucun bundle à maintenir côté client
  • Chargement ultra-rapide : parsing JS minimal, pas de virtual DOM, rendu natif
  • Maintenabilité maximale : tout développeur JS peut lire et modifier le code sans formation framework
  • Modularité fonctionnelle : 65 fichiers JS organisés par domaine (contacts/, contracts/, modules/SEO/, etc.)

Deno 2.x — Configuration Edge Functions

Toutes les Edge Functions utilisent Deno 2.x avec Deno.serve() natif (sans import std/http/server). Un fichier import map central (supabase/functions/deno.json) gère les dépendances :

supabase/functions/deno.json (import map Deno 2.x) { "imports": { "@supabase/supabase-js": "npm:@supabase/supabase-js@2", "stripe": "npm:stripe@14", "pdf-lib": "npm:pdf-lib", "@std/encoding/base64": "jsr:@std/encoding@^1.0.0/base64" } } Avantage : bare imports dans les sources → découplage entre alias et fournisseur npm/jsr. CI/CD : deno check --config supabase/functions/deno.json $file (GitHub Actions)

Sécurité technique

  • JWT : Supabase Auth émet des JWT signés RS256, vérifiés à chaque requête Edge Function via getUser()
  • RLS : Row Level Security activé sur les 28 tables — aucun accès possible sans JWT valide
  • Secrets : clés API stockées dans Supabase Vault (variables d'environnement Edge Functions), jamais exposées côté client
  • CORS : Edge Functions limitées à https://crm.safe-digitalisation.fr (origin whitelist)
  • Anti-bruteforce : Edge Function record-login-failure + tables login_attempts + rate_limits
  • TOTP MFA : Google Authenticator / Authy — code 6 chiffres, trust 2h en localStorage, enrôlement forcé roles sensibles
  • Chiffrement : TLS 1.2+ en transit, AES-256 au repos (Supabase)
  • Hachage OTP : SHA-256 pour signatures mandats, stocké hashé — non réversible

Mécanisme de dépassement volontaire (friction intentionnelle)

Si un commercial souhaite dépasser sa limite, il doit traverser 4 étapes de sensibilisation avant de pouvoir forcer le calcul. Ce mécanisme de friction intentionnelle rend le dépassement possible mais non anodin.

Étape 1 — Alerte
Message d'information : distance calculée, CO₂ estimé, limite du rôle. Bouton "Je comprends" requis.
Étape 2 — Impact
Équivalence concrète du CO₂ excédentaire (arbres à planter, km en train équivalent). Checkbox "J'ai lu l'impact" requise.
Étape 3 — Alternatives
Suggestions : réduire le rayon, scinder en 2 tournées, regrouper les prochaines visites. Bouton "J'ai considéré les alternatives".
Étape 4 — Confirmation
Bouton rouge "Forcer quand même — Je dépasse la limite éco". Le dépassement est tracé dans la base (force_depassement = true).

Maximum 5 étapes par tournée

Contrainte structurelle enforced en base de données (CHECK nb_etapes <= 5) et dans l'interface. Limite la durée des tournées et réduit l'épuisement des commerciaux.
